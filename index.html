DoSomething.org: A year ago

  • Upgrades: Drupal 5 > Drupal 6 > Drupal 7
  • Technical debt out the wazoo
  • Poor performance
  • Loosely enforced development practices
Talk a bit about the state of the engineering team, dev process and status of the codebase.

Problem: Outsourced development

  • Black-box products thrown “over the wall”
  • No institutional knowledge
  • Decreased visibility → increased technical debt

Problem: Loose dev & deploy workflow

Problem: Lack of Drupal knowledge

  • “I wish Drupal had a theming layer.”
  • 250+ modules
  • Missing content strategy

Problem: Unhappy developers

  • “I spent 90% of my time wrestling with the platform and fixing problems.”
  • Technical debt → Unhappiness
  • No way to attract talent!
  • No way to retain talent!

Speak about the process and formula for introducing change

Destroy the Silos: Deets

  • Think lean: Make one testable change at a time
  • Enforce process improvements around new development
  • Code review: Github pull requests
  • Asynchronous chat: Hipchat
  • Org-accessible issue tracking system: Trello
Speak briefly about how there was not much collaboration between developers. Commonplace for employees to stop by a developer's desk with some issue or task. Issues were discussed via email threads.

Automate The Things

Speak a bit about Hubot, Jenkins and automating deployments. Also about automating useful tasks.

What's Next?

  • Improving test coverage
  • Improving continuous integration
  • Vagrant and Salt Stack (No more “It works for me”)
  • Better alerts and logging
  • Issue tracking improvements (Trello-specific since it does lack many things)
  • Externalize tech's bandwidth constraints (Kanban style)
  • More decision power to individual devs: Matrix teams
